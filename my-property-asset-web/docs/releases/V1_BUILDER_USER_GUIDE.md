# V1 Builder User Guide

## Access

Sign in as Builder Owner/Admin → `/builder-portal`.

## Core Flows

1. **Dashboard** — delivery KPIs, branding/comms/subscription summaries.  
2. **Company** — organization profile.  
3. **Projects** — create/manage projects; hierarchy/buildings/units.  
4. **Owners** — prospects, assignments, invitations.  
5. **Documents** — metadata upload (V1 does not stream arbitrary binaries).  
6. **Handovers** — digital handover packages.  
7. **Branding** — white-label identity for Owner App contract.  
8. **Communications** — announcements to owners.  
9. **Subscription** — plan usage, renew, upgrade (Owner); view (Admin).

## Plan Limits

If a limit is reached (projects, units, storage, notifications), the portal shows an upgrade prompt and routes to Subscription → Plans.

## Roles

| Role | Typical access |
|---|---|
| Owner | Full builder ops + subscription renew/upgrade |
| Admin | Operational access; subscription view-only |
| Staff | Operational contribute where permitted; no subscription |

## Support

Contact platform support via the email/phone configured by Super Admin, or open a ticket through Support Center (Super Admin managed).
