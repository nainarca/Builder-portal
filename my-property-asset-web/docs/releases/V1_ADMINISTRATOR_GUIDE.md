# V1 Administrator Guide (Super Admin)

## Access

Sign in with a Super Admin account → `/super-admin`.

## Daily Operations

1. **Dashboard** — builders, trials, projects, health, support queue, recent audit.  
2. **Builders** — activate, suspend, reactivate, soft-delete; review profile/activity.  
3. **Billing** — plans, assign/extend trial, suspend subscription, invoices.  
4. **Branding Oversight** — reset, disable, restore defaults.  
5. **Operations** — health, audit, activity, monitoring, alerts, communications moderation.  
6. **Support** — ticket queue, assign, notes, resolve.  
7. **Analytics** — growth charts (revenue placeholder).  
8. **Settings** — platform name, support contacts, legal URLs, maintenance mode, feature flags.

## Emergency

1. Enable **Maintenance Mode** in Platform Settings.  
2. Suspend affected builder or subscription.  
3. Disable abusive communications.  
4. Follow `V1_ROLLBACK_GUIDE.md` if needed.

## Do Not

- Grant Builder Staff access to Super Admin.  
- Apply incomplete SQL (handover before units/owners).  
- Treat demo mock data as production truth.
